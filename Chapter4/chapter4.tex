This chapter describes in detail and compares created algorithms for the VCR property detection.

\section{Overview}
Detection algorithms are used to check whether election profile satisfies desirable structural properties.
For example to check CR (VR) properties it's enough to check for the \textit{Consecutive Ones Property} in the preferences matrix.

My focus lies on the one dimmensional VCR property. 
Having preferences matrix as input we have to provide mapping of each agent (voter/candidate) to the point on real line
and nonnegative number representing radius, so that it satisfies the VCR property.  

\section{Integer Linear Programming}
Solution based on the LP is excellent choice for the first algorithm to develop. 
It provides efficient, correct (directly based on the VCR property definition) and relatively simple detection algorithm,
Additionally it can be used as a benchmark for checking future solutions.

The only input is \textit{approval set} of each voter. Represented by $2D$ matrix $A_{vc}$

\subsubsection{Variables}

$x_c$ - candidate $c$ position on $x$ axis \\
$x_v$ - voter $v$ position on $x$ axis \\
$r_c$ - candidate $c$ radius \\
$r_v$ - voter $v$ radius 

\subsubsection{Transform VCR condition to linear constraints}

VCR definition formula contains square root and power elements (WIP: Inne słowo).
Hence it's not directly suitable for ILP formulation. Let's transform it to linear constraint.

\begin{equation}
    \begin{split}
    VCR & = \\
     & = \sqrt{(x_c - x_v)^2} \le r_c + r_v \\
     & = \|x_c - x_v\| \le r_c + r_v \\
     & = -(r_c + r_v) \le x_c - x_v \le r_c + r_v \\
     & \iff x_c - x_v \le r_c + r_v\ \ \lor \ \ -(x_c - x_v) \le r_c + r_v
    \end{split}
\end{equation}

\subsubsection{Indicatior Constraint}
Above constraint contains $\lor$ logical condition which cannot be represented directly in the linear programming.
However the issue is quite common and most solvers provide additional constraint types to solve it.
Gurobi solver provides \textit{IndicatorConstraint} which allows to create versatile statements like $if x==5 than y > 5 else y < 2$
This approach is losely based (WIP Fact check) on the so called \textit{BigM} method which is low level solution, and requires parameter tunning depending on the problem.
