\section{Basic Definitions}

\subsubsection{Approval Elections}

An approval-based election is a pair $E = (C, V)$, where 
$C = \{c_1,...,c_M\}$ is a set of candidates and  $V = \{v_1,...,v_N\}$ is a set of voters.
We use $M$ and $N$ for the number of candidates and voters respectively. 
For each voter $v \in V$, his or hers preferences are represented as a set of approved candidates $A(v)$,
we call it an \textit{approval set}.
Furthermore, for each candidate $c \in C$, we define $V(c) = \{v \in V \:|\: c \in A(v)\}$,
a set of voters who approve candidate $c$.

In practice, we often use an additional, simplified notation of representing an election.
We redefine $A$ as a two-dimensional binary matrix, with $N$ rows and $M$ columns,
where value of $A[i][j]$ is $(1)$ if i-th voter approves of j-th candidate, and $(0)$ otherwise.
In other words, each row of the matrix is a vector representation of a given voter's approval set.
We refer to it as an \textit{approval matrix} or a \textit{preference matrix}.
It will always be clear from the context which notation we are using.

\subsubsection{Committee Selection Rules}

Voting rules are algorithms used to, select a committee (winners) — fixed size subset of the candidates,
based on the voters preferences, for a given election.
Each rule has different properties and must be selected accordingly to the goal of the election.
As this is not the main focus of this work we point to \cite{faliszewskiSkowronMultiWinnerOverview},
for the detailed overview of multiwinner voting rules.

Nonetheless, we introduce the most basic voting rule for the approval model, simply called \textit{Approval Voting (AV)},
introduced by \cite{brams1983approval}.

\begin{definition}[Multi-Winner Approval Voting (AV)]
For a given election and a size of committee $k$, AV rule selects the $k$ candidates with the highest approval score,
which, for each candidate $c$, is the number of voter who approve of $c$.
\end{definition}

Many social choice problems such as winner determination, or control problem are known to be computationally hard.

\subsection{Euclidean Preference Restrictions}

\subsubsection{Voter Candidate Range (VCR)}\label{VCR-definition}
Given an election \(E = (C, V)\), we say that the voters have \textit{1D-Euclidean} preferences if
for every agent (voter or candidate) there exists a point $x_a$ and nonnegative real value radius $r_a$ such that:
$\forall \text{ } c \in C \text{ } \forall \text{ } v \in V: c \in A(v) \Leftrightarrow \sqrt{(x_c - x_v)^2} \leq r_c + r_v$

We can easily think of it as: the point $x_a$ represents ideal position 
in the 1-dimensional space of opinions. The radius can be seen as candidate's charisma or
voter's willingness to compromise.

\subsubsection{Candidate Range (CR)}
We say that P satisfies CR if candidates in $C$ can be
ordered so that each of the sets $v_i$ forms an interval of that ordering.
CR is special case of VCR, where we require all the voters to have radii equal to zero

\subsubsection{Voter Range (VR)}
We say that P satisfies VR if the voters in $V$ can be reordered
so that for every candidate $c_i$ the voters that approve c form an interval of that
ordering.
VR is a special case of VCR, where we require all the candidates to have radii equal to zero

\subsection{Integer Linear Program}
Integer linear programming (ILP) is a common approach to solve NP-hard problems, 
furthermore it can be used effectively for other problems as well.

we have to create a system of linear inequalities (constraints)
